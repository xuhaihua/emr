样式表

在EMR中它是一个json文件，用于存诸对元素样式统一设置的列表

  • 样式选择器

id选择器

语法 : #id属性值
作用 : 选中对应id属性值的元素

类选择器

语法 : .stylesheet属性值
作用 : 选中对应stylesheet属性值的元素

name选择器

语法 : $name属性值
作用 : 选中对应name属性值的元素

元素选择器

语法 : 标签名（注意此处的标签名一定是该节点类型的完全限定名）
作用 : 选中对应标签中的内容

  • 选择器优选级

  1. id 选择器
  2. 类选择器
  3. name选择器
  4. 元素选择器

Example：

以下代码通过样式文件给视图中的节点设置外观样式

样式文件 

/*
* 样式表
*/
{
    // id选择器 （给标签id属性为panelImg的这个节点加上一个背景图）
    "#panelImg": {
        "backgroundImage": "sources/image/pic"
    },

    // 类选择器 （背景色为红色）
    ".bgRed": {
        "backgroundColor": "255,0,0"
    },

    // 类选择器 （背景色为红色）
    ".bgGreen": {
        "backgroundColor": "0,255,0"
    },

    // name选择器（所有标签name属性为panel的节点宽度为：200mm、高度为：100mm，x,y的坐标分别为0,1500mm）
    "$panel": {
        "x": 0,
        "z": 1500,
        "width": 200,
        "height": 100
    },

    // 标签选择器 （所有PanelRoot标签背景色统一为白色 【注意：这里的标签名为节点类型的完全限定名】）
    "EMR.Entity.PanelRoot": {
        "backgroundColor": "255,255,255"
    }
}

在视图中引入一个样式表通过Stylesheet标签，其中path属性指向样式表所在的Resources路径，一般该标签见意放在Root下的第一行

  • EMR样式表与html中样式表对比

  1. 与html css样式表一样一个视图文件可以引入多个样式表，通过添加多个Stylesheet标签完成，它们都是层叠式的。
  2. EMR中的样式表是一个json文件而html中的样式表是一个以.css结尾的文件。
  3. EMR中的样式表在这一版中只有以上这四个选择器，没有其它的选择器而html css中的选择器更多功能更强大，例如：后代选择器、子代选择器、相邻兄弟选择器等并且支持通配符而这些在EMR样式表中不存在。
  4. EMR的样式不存在继随特性例如：在父元素上的属性fontSize的值不会被它的子元素继承至少在这一版中是这样后续会改进，而html css中的样式是可以被子元素继承的。
