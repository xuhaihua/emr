Collider (碰撞体)

collider在unity中是一个碰撞体，它用于描述虚拟物体的碰撞边界。在EMR中有很多节点事件是依赖于collider的，例如：单击事件、触摸事件、节点碰撞事件等，只有拥有collider的节点才能触发这些事件。

上图在空间中放置了一个节点，图中这些绿色线条包围的区域就是这个节点的collider，它描述了该节点的碰撞边界，该collider的大小、位置与该节点重合。

  • 检查节点是否已有collider

可以通过hasCollider属性检查一个节点是否已包含collider

hasCollider属性:

hasCollider 节点是否已包含collider bool -

  • 通过节点属性控制collider

可以通过节点的collider属性，控制节点如何添加collider，当该属性值为true时节点会自动创建一个collider并且这个collider会被一直保持不被回收掉，否则会自动按照该节点的实际情况创建和回收collider。

collider属性:

collider 是否强制添加collider，如果为: true则该节点会自动创建一个collider并且这个collider会被一直保持不被回收，否则会按照该节点的实际情况创建和回收这个collider（例如：该节点注册有单击事件、触摸事件时，会自动给这个节点带上collider，当这些事件都被注销后回收这个collider） bool 默认为: false

  • 手动添加、查找和移除collider

除了通过节点的collider属性控制节点如何添加collider外，还可以在节点上通过以下方法手动操作节点的collider

名称 说明
BoxCollider addCollider (Vector3 center, Vector3 size, string name) 向节点添加一个collider，其中参数center为：collider的中心点，size为：collider的尺寸（单位为：比例），name为：collider的名称
BoxCollider findColliderByName (string name) 按名称查找collider
void removeCollider (string name) 移除节点中指定名称的collider

由节点属性添加的collider，无法通过removeCollider方法移除，因为该collider的添加和移除只受系统控制
